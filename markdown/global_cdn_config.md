# 全球最优延迟选择配置

## 概述

Zero平台提供全球落地节点智能调度功能，支持实时路径监测与自动切换，确保全球网络访问的最低延迟路径。系统通过动态路由算法，为每个连接请求智能选择最优的落地节点。

## 应用场景

在分布式网络架构中，用户在香港部署入口服务器，同时在美国、日本、新加坡、智利、德国、英国等多个地区部署落地节点服务器。

## 功能特性

- **智能路由选择**：系统根据实时网络延迟动态选择最优落地节点
- **地理位置匹配**：访问美国网站自动使用美国节点，访问香港网站自动使用香港节点
- **故障自愈机制**：节点故障或延迟异常时，20秒内自动切换到备用节点
- **高可用保障**：确保网络连接的持续可用性和最佳性能

**系统要求：** 版本 >= 1.0.31

## 配置流程

### 第一步：创建全球落地节点组

将所有全球落地节点服务器添加到同一个服务器组中，实现统一管理和调度。

### 第二步：配置端口转发和转发链

创建端口转发规则，设置入站代理，并配置多级转发链路：

<img src="https://img.coderluny.com:444/uploads/2e59d01e-9531-443b-99be-1c36e857313b.png" width="600" alt="端口转发和转发链配置">

### 关键配置要点

1. **启用转发链模式**：确保数据能够通过多级节点进行转发
2. **设置全球落地为终端级**：'全球落地'节点组必须配置为转发链的最后一级
3. **开启最佳延迟模式**：启用智能路由算法，实现最优节点自动选择
4. **配置入站代理参数**：在高级选项中添加以下JSON配置

```json
{
  "Ss": {}
}
```

<img src="https://img.coderluny.com:444/uploads/305969df-58a5-428b-b95a-4c8c0efb8a61.png" width="600" alt="高级选项入站代理配置">

## 配置效果

完成上述配置后，通过入站代理的SS节点访问全球网站时，系统将：

- **自动选择最优节点**：从'全球落地'节点组中智能选择延迟最低的节点
- **实现故障自愈**：当前节点链路中断时，20秒内自动切换到备用节点
- **保障服务连续性**：确保网络连接始终保持可用状态
