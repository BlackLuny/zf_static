核心概念

1. 转发端点

转发端点是Zero Forwarder的核心组件，为服务器和端口转发提供强大的数据转发能力，支持端到端加密和0-RTT快速连接技术。

主要功能：

  • 服务器转发：在添加服务器时选择转发端点，所有服务器与控制器之间的流量将通过转发端点进行安全转发
  • 统一传输：启用"使用转发端点作为传输"选项后，该服务器下的所有端口转发也将统一使用相同的转发端点
  • 端口级控制：在添加单个端口时可独立选择专用的转发端点，实现精细化流量管理
  • 自定义配置：支持用户自定义添加转发端点，灵活适配不同网络环境
  • 数据聚合：转发端点可作为Tot服务器使用，聚合多条转发链路的数据流，实现带宽叠加效果

2. Tot服务器

Tot服务器专门用于数据流聚合和优化，通过智能合并多条转发端点的数据流，最终统一转发至目标地址，有效提升传输效率和网络稳定性。

3. 负载均衡模式

根据不同的网络需求和场景，提供四种智能负载均衡策略：

  • 最优延迟：实时监测网络状态，自动选择总延迟最低的转发路径。系统会动态评估所有可用的转发端点和目标，智能切换至最优线路
  • 故障转移：采用主备模式，优先使用首选转发目标。当主线路出现故障时自动切换到备用线路，主线路恢复后立即回切，确保服务连续性
  • 随机选择：在多个转发目标中随机分配连接，有效分散负载，适用于流量均衡场景
  • 轮询调度（RoundRobin）：按顺序轮流选择转发目标，确保各条线路负载均匀分布

4. 高可用架构

通过"多转发端点 + 多转发目标"的组合配置，构建企业级高可用网络架构：

  • 多路径冗余：同时配置多个转发端点，消除单点故障风险
  • 多目标备份：设置多个转发目标，确保服务持续可用
  • 智能切换：结合负载均衡模式，实现毫秒级故障检测和自动切换
  • 带宽聚合：利用多条线路同时传输，大幅提升网络吞吐量
